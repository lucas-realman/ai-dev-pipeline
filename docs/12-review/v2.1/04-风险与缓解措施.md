> **评审编号**: REVIEW-v2.1-04  
> **评审范围**: 系统技术风险、安全风险、运维风险识别及缓解方案评估  
> **评审日期**: 2026-03-07  
> **评审依据**: 源码分析 + 01~03 评审发现  
> **评审结论**: 已识别 15 项风险，P0 风险 3 项均有缓解路径

## §1 风险分级标准

| 等级 | 概率 × 影响 | 说明 |
|------|-----------|------|
| **P0 — 严重** | 高概率 × 高影响 | 可能导致系统不可用或数据丢失 |
| **P1 — 重要** | 中概率 × 高影响，或高概率 × 中影响 | 影响核心功能或用户体验 |
| **P2 — 一般** | 低概率 × 高影响，或中概率 × 中影响 | 特定场景下可能出现问题 |
| **P3 — 观察** | 低概率 × 低影响 | 未来可能需要关注 |

## §2 单点故障风险

### RISK-001: Orchestrator 单实例崩溃 [P0]

| 属性 | 值 |
|------|---|
| **触发条件** | Python 进程崩溃、OOM、服务器宕机 |
| **影响** | 所有运行中任务中断，内存中状态全部丢失，需完整重跑 |
| **当前缓解** | systemd restart + 日志记录 |
| **缺失缓解** | 状态持久化、断点恢复 |
| **建议** | (1) 实现 JSON 快照持久化 (每状态变更写盘) — 2h; (2) 启动时加载快照恢复 — 2h |

### RISK-002: LLM API 长时间不可用 [P1]

| 属性 | 值 |
|------|---|
| **触发条件** | OpenAI 兼容 API 服务中断、限流、密钥过期 |
| **影响** | doc_analyzer 返回空任务列表 (Sprint 无法启动); reviewer L2/L3 全部降级通过 |
| **当前缓解** | reviewer 降级通过策略 (score=4.0) |
| **缺失缓解** | doc_analyzer 无降级; 无限流重试; 无备用 LLM |
| **建议** | (1) _call_llm 增加 3 次指数退避重试 — 2h; (2) 支持备用 LLM endpoint 配置 — 1h |

### RISK-003: SSH 连接批量失败 [P1]

| 属性 | 值 |
|------|---|
| **触发条件** | 网络分区、SSH 密钥轮换、目标机 sshd 崩溃 |
| **影响** | 所有 dispatch 失败，任务进入 RETRY → ESCALATED |
| **当前缓解** | ConnectTimeout=10s + 任务级重试 (max_retries=3) |
| **建议** | (1) 连接预检: dispatch 前 ssh ping — 1h; (2) 机器自动标记 OFFLINE — 1h |

## §3 数据完整性风险

### RISK-004: Git 推送冲突升级 [P1]

| 属性 | 值 |
|------|---|
| **触发条件** | 多台机器同时完成编码并推送同一分支 |
| **影响** | rebase/merge 失败，3 次重试耗尽后 push 失败，任务标记失败 |
| **当前缓解** | 3 层重试: rebase → no-rebase → sleep+retry |
| **评估** | 4 台机器场景下冲突概率约 15-20%，现有重试策略足够 |
| **建议** | (1) 当机器数 > 8 时引入 merge queue; (2) 考虑每机器独立分支 + 最终合并 |

### RISK-005: 状态不一致 — MachineRegistry vs TaskEngine [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | `mark_dispatched()` 成功设置机器 BUSY，但 SSH 执行立即失败 |
| **影响** | 机器标记为 BUSY 但无任务运行，后续调度跳过该机器 |
| **当前缓解** | `handle_coding_done()` 会 `set_idle()` 释放机器 |
| **评估** | 如果 dispatch_task 异常未被 handle_coding_done 处理，机器永久 BUSY |
| **建议** | 在 main loop 中增加 stale-busy 检测 (busy > 2×timeout 则强制释放) — 1h |

### RISK-006: 循环依赖死锁 [P0]

| 属性 | 值 |
|------|---|
| **触发条件** | LLM 生成的任务列表中 A depends_on B, B depends_on A |
| **影响** | `next_batch()` 永远无法满足依赖条件，Sprint 永不结束 |
| **当前缓解** | ❌ 无 (ALG-026 已设计但未实现) |
| **建议** | enqueue() 时执行拓扑排序 + 环路检测，发现环路立即 ESCALATE — 2h |

## §4 安全风险

### RISK-007: SSH 脚本注入 [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | task.description 或 task.target_dir 包含 shell 特殊字符 (`;`, `$()`, `` ` ``) |
| **影响** | 远程执行恶意命令 |
| **当前缓解** | 脚本通过 stdin pipe 发送 (非命令行参数)，降低了注入风险 |
| **评估** | 风险较低 — 任务描述来自 LLM 解析，非用户直接输入 |
| **建议** | 对 task_id / target_dir 做字符白名单校验 — 0.5h |

### RISK-008: API Key 泄露 [P1]

| 属性 | 值 |
|------|---|
| **触发条件** | OPENAI_API_KEY 硬编码在配置文件或脚本中 |
| **影响** | 密钥泄露导致 LLM 资源被滥用 |
| **当前缓解** | config.yaml 使用 `${OPENAI_API_KEY}` 引用环境变量 ✅ |
| **隐患** | `_build_ssh_script()` 将 API key 写入远程执行脚本 → 远端 /proc 可能暴露 |
| **建议** | 通过 SSH 环境变量传递而非脚本内嵌 — 1h |

### RISK-009: 钉钉 Webhook Token 泄露 [P3]

| 属性 | 值 |
|------|---|
| **触发条件** | webhook URL 包含 access_token，日志可能记录完整 URL |
| **影响** | 任何人可向群组发消息 |
| **当前缓解** | HMAC 签名 + secret 增加安全层 ✅ |
| **建议** | 确保日志脱敏 (mask access_token 后 8 位) — 0.5h |

## §5 可靠性风险

### RISK-010: aider 版本不兼容 [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | aider CLI 升级后参数变化 (如 --yes-always 移除) |
| **影响** | 所有编码任务失败 |
| **当前缓解** | ❌ 无版本锁定 |
| **建议** | (1) 在 config.yaml 中配置 aider 版本; (2) dispatch 前检查 `aider --version` — 1h |

### RISK-011: LLM 输出质量退化 [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | 模型更新、API provider 切换模型版本 |
| **影响** | doc_analyzer 分解出低质量任务; reviewer 评分不准确 |
| **当前缓解** | Prompt 固定 + temperature 低 (0.1-0.2) |
| **建议** | (1) 保存 LLM 请求/响应日志用于回溯; (2) baseline 测试集 — 4h |

### RISK-012: pytest-json-report 缺失 [P3]

| 属性 | 值 |
|------|---|
| **触发条件** | 目标机未安装 pytest-json-report 插件 |
| **影响** | JSON 报告不生成，回退到 stdout 解析 (精度降低) |
| **当前缓解** | 双通道解析: JSON 优先 → stdout 回退 ✅ |
| **建议** | 部署脚本中加入 `pip install pytest-json-report` — 0.5h |

## §6 运维风险

### RISK-013: 日志无结构化 [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | 生产环境需日志分析和告警 |
| **影响** | 无法接入 ELK/Grafana，故障排查依赖 grep |
| **当前缓解** | loguru 文本日志 |
| **建议** | 切换为 JSON 格式日志 (loguru serialize=True) — 2h |

### RISK-014: 无监控指标 [P2]

| 属性 | 值 |
|------|---|
| **触发条件** | 需要实时监控系统健康度 |
| **影响** | 无法提前发现异常趋势 (如 SSH 超时增加、LLM 延迟上升) |
| **当前缓解** | ❌ 无 |
| **建议** | 暴露 Prometheus 指标: 任务成功率、SSH 延迟 p99、LLM 调用耗时 — 4h |

### RISK-015: 无灰度/回滚机制 [P3]

| 属性 | 值 |
|------|---|
| **触发条件** | Orchestrator 升级后行为异常 |
| **影响** | 需手动回退代码 + 重启 |
| **当前缓解** | Git tag 标记版本 ✅ |
| **建议** | 部署脚本支持 rollback 命令 — 2h |
