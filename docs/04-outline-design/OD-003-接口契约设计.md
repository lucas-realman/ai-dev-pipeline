### 1.4 IF-004: TaskEngine.next_batch

```python
def next_batch(self, registry: MachineRegistry) -> List[CodingTask]:
    """
    取出下一批可并行执行的任务。

    逻辑:
    1. 从 QUEUED 任务中筛选依赖已满足的
    2. 从 registry 获取 idle 机器
    3. match_machine(task.tags) 匹配
    4. 填入 task.assigned_machine
    5. 最多返回 concurrent_limit 个任务

    Returns:
        已分配机器的可执行任务列表 (可能为空)
    """
```

### 1.5 IF-005: MachineRegistry.match_machine

```python
def match_machine(
    self,
    required_tags: List[str],
    candidates: Optional[List[MachineInfo]] = None
) -> Optional[MachineInfo]:
    """
    从候选机器中匹配标签 + 最低负载的机器。

    匹配规则:
    1. 候选机器须包含 required_tags 中所有标签
    2. 多个候选取 cpu_percent 最低的
    3. 如果 required_tags 为空，匹配所有 idle 机器

    Returns:
        最佳匹配的机器，或 None
    """
```

### 1.9 IF-010: TaskStateMachine 转换方法

```python
class TaskStateMachine:
    def enqueue(self) -> None          # CREATED → QUEUED
    def dispatch(self) -> None         # QUEUED → DISPATCHED
    def coding_done(self, r: TaskResult) -> None    # → CODING_DONE / RETRY / ESCALATED
    def start_review(self) -> None     # CODING_DONE → REVIEW
    def review_done(self, r: ReviewResult) -> None  # → TESTING / RETRY / ESCALATED
    def test_done(self, r: TestResult) -> None      # TESTING → JUDGING
    def judge(self, r: TestResult) -> None          # → PASSED / FAILED
    def handle_failure(self) -> None   # FAILED → RETRY / ESCALATED
    def requeue(self) -> None          # RETRY → QUEUED

    # 非法转换时 raise StateMachineError
```

## §2 外部 API 接口

> 映射: SYS-007 (通知), ADR-004 (LLM 分工)

### 2.1 LLM API (OpenAI 兼容)

| 属性 | 值 |
|------|---|
| **端点** | `POST {OPENAI_API_BASE}/chat/completions` |
| **认证** | `Authorization: Bearer {OPENAI_API_KEY}` |
| **模型** | `openai/claude-opus-4-6` |
| **调用方** | doc_analyzer (文档拆解), reviewer L2/L3 (审查) |
| **超时** | 120s |

**请求体**:
```json
{
  "model": "openai/claude-opus-4-6",
  "messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}],
  "temperature": 0.2,
  "response_format": {"type": "json_object"}
}
```

### 2.2 钉钉 Webhook API

| 属性 | 值 |
|------|---|
| **端点** | `POST https://oapi.dingtalk.com/robot/send?access_token={token}` |
| **认证** | URL 含 access_token + 签名 (timestamp + secret HMAC-SHA256) |
| **调用方** | reporter.py |
| **消息类型** | Markdown |

**请求体**:
```json
{
  "msgtype": "markdown",
  "markdown": {
    "title": "Sprint 1 — 任务 #3 升级为人工",
    "text": "### 任务升级\n- task_id: S1-003\n- 失败原因: Review L3 评分 2.1/5.0\n- 修复建议: ..."
  },
  "at": {"atMobiles": ["138xxxx"], "isAtAll": false}
}
```

### 2.3 SSH 远程执行协议

| 属性 | 值 |
|------|---|
| **协议** | SSH (OpenSSH 兼容) |
| **认证** | 密钥认证 (ed25519) |
| **端口** | 22 (默认) |
| **操作序列** | ① scp 指令文件 → ② ssh exec aider → ③ 收集结果 |

**aider 命令模板**:
```bash
cd {work_dir} && \
aider --model openai/claude-opus-4-6 \
      --openai-api-base {OPENAI_API_BASE} \
      --openai-api-key {OPENAI_API_KEY} \
      --message-file /tmp/aider_msg_{task_id} \
      --yes-always \
      --auto-commits \
      --no-suggest-shell-commands \
      {context_files...}
```

## §3 错误码与异常约定

| 错误类型 | 抛出模块 | 处理方式 |
|----------|---------|---------|
| `StateMachineError` | state_machine | 非法状态转换，日志 + 升级 |
| `httpx.HTTPError` | doc_analyzer, reviewer | 重试 1 次 → 失败则任务 ESCALATED |
| SSH 连接超时 | dispatcher | 标记机器 ERROR → 任务迁移 |
| aider exit_code ≠ 0 | dispatcher | 进入 RETRY 流程 |
| pytest 找不到测试文件 | test_runner | 默认 PASSED (auto-pass) |
| JSON 解析失败 | doc_analyzer | 重试 1 次 → 失败则 ESCALATED |
