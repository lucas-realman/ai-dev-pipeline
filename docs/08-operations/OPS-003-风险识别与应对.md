## §2 应对策略

| 风险 ID | 缓解措施 | 应急预案 |
|---------|---------|---------|
| RISK-001 | httpx 重试 2 次 + 指数退避; 可切换备用 LLM | 降级为纯 L1 静态审查，跳过 L2/L3 |
| RISK-002 | asyncssh 超时 30s + 自动迁移到 idle 机器 | 手动 SSH 诊断; requeue 任务 |
| RISK-003 | machine_registry 心跳检测 → 自动标 ERROR | 任务 requeue → 其他机器接管 |
| RISK-004 | pyproject.toml 锁定 aider-chat 版本 | 回退到已知可用版本 |
| RISK-005 | 文档驱动减少隐性知识; 钉钉报告自动化 | 关键操作脚本化 (scripts/) |
| RISK-006 | 三层 Review 交叉验证; temperature=0.2 | 升级为人工 Review (ESCALATED) |
| RISK-007 | 监控磁盘 > 80% 告警 | git gc --aggressive |
