## §1 术语定义

| 术语 | 英文 | 定义 |
|------|------|------|
| **Sprint** | Sprint | 一次完整的迭代周期：文档拆解 → 编码 → 审查 → 测试 → 报告 |
| **Round** | Round | Sprint 内的一个批次循环，每轮处理一批并行任务 |
| **任务卡** | Task Card | 文档中描述的一个独立编码任务，对应一个 `CodingTask` 实例 |
| **薄调度层** | Thin Orchestration | orchestrator 只做调度不做业务逻辑，编码全部下发到工作节点 |
| **文档驱动** | Document-Driven | 以需求/设计文档为输入，LLM 自动拆解为编码任务 |
| **短路失败** | Short-Circuit | 审查中某一层失败即立即返回，不继续后续层 |
| **升级** | Escalation | 任务重试超限后标记为 ESCALATED，通知人工介入 |
| **fallback 阈值** | Fallback Threshold | 测试通过率低于此值仍判 PASSED 的容忍比例 |
| **标签匹配** | Tag Matching | 根据任务 tags 与机器 tags 的交集进行调度 |
| **红绿灯报告** | Traffic Light Report | 用红(失败)/绿(通过)/黄(警告)标记的 Sprint 摘要 |
