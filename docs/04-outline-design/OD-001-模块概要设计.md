## §1 模块清单与职责

### 1.1 MOD-001: DocAnalyzer (doc_analyzer.py, 291 行)

| 属性 | 值 |
|------|---|
| **核心类** | `DocAnalyzer` |
| **ARCH** | ARCH-001 文档解析组件 |
| **FR** | FR-001 文档集加载, FR-002 AI 自动拆解, FR-003 结构化任务输出 |
| **职责** | 按 doc_set glob 模式加载项目文档 → 调用 LLM 进行自动任务分解 → 输出 `List[CodingTask]` |
| **依赖** | config (LLM 配置), task_models (`CodingTask`) |
| **对外接口** | `load_doc_set() → Dict[str, str]`, `analyze_and_decompose(sprint_id) → List[CodingTask]` |

**核心流程**:
1. `load_doc_set()` — 按 glob 模式 (`docs/01-*.md` 等) 收集并拼接文档内容
2. `_build_decompose_prompt()` — 构造 LLM prompt，含文档全文 + 拆解约束
3. `_call_llm()` — 调用云端 LLM API (Claude Opus 4.6)，获取 JSON 响应
4. `_parse_tasks()` — 将 JSON 转换为 `CodingTask` 列表

### 1.6 MOD-006: Dispatcher (dispatcher.py, 325 行)

| 属性 | 值 |

### 1.7 MOD-007: AutoReviewer (reviewer.py, 262 行)

| 属性 | 值 |
|------|---|
| **核心类** | `AutoReviewer` |
| **ARCH** | ARCH-004 审查组件 |
| **FR** | FR-009 L1 静态检查, FR-010 L2 契约对齐, FR-011 L3 设计符合度 |
| **职责** | 三层自动 Review：L1 py_compile+ruff → L2 LLM 契约检查 → L3 LLM 设计检查 |
| **依赖** | config, task_models |
| **对外接口** | `review(task, result) → ReviewResult` |

**三层 Review 策略**:

| 层 | 名称 | 工具 | 成本 | 通过率 |
|----|------|------|------|-------|
| L1 | 静态检查 | py_compile + ruff | 0 (本地) | ~95% |
| L2 | 契约对齐 | LLM (Claude) | API 调用 | ~85% |
| L3 | 设计符合度 | LLM (Claude) | API 调用 | ~90% |

## §2 模块间接口一览

> 映射: ARCH-001~010 → IF-001~012 (详见 [OD-003](OD-003-接口契约设计.md))

### 2.1 核心调用链

```
main.Orchestrator.run_sprint()
│
├── 1. DocAnalyzer.analyze_and_decompose(sprint_id)
│       ├── load_doc_set() → Dict[str, str]
│       ├── _call_llm(prompt) → JSON
│       └── return List[CodingTask]
│
├── 2. TaskEngine.enqueue(tasks)
│       └── _topological_sort(tasks)
│
├── 3. [LOOP] TaskEngine.next_batch(registry)
│       ├── MachineRegistry.get_idle_machines()
│       ├── MachineRegistry.match_machine(tags)
│       └── return List[CodingTask] (assigned_machine 已填)
│
├── 4. Dispatcher.dispatch_batch(tasks)
│       ├── [PARALLEL] dispatch_task(task)
│       │   ├── _build_instruction(task) → str
│       │   ├── _scp_content(machine, instruction)
│       │   ├── _ssh_exec(machine, aider_cmd) → stdout, exit_code
│       │   └── return TaskResult
│       └── return List[TaskResult]
│
├── 5. AutoReviewer.review(task, result) → ReviewResult
│       ├── L1: _static_check(files) — py_compile + ruff
│       ├── L2: _contract_check(task, files) — LLM
│       └── L3: _design_check(task, files) — LLM
│
├── 6. TestRunner.run_tests(task) → TestResult
│       ├── _find_tests_for_task(task)
│       ├── _exec(pytest_cmd)
│       └── _parse_json_report() / _parse_pytest_output()
│
├── 7. TaskStateMachine.transition(event)
│       ├── coding_done() / review_done() / test_done()
│       ├── judge() → PASSED / FAILED
│       └── handle_failure() → RETRY / ESCALATED
│
├── 8. Reporter.notify(...)
│       ├── _send_dingtalk_webhook(markdown)
│       └── _save_local_report(markdown)
│
└── 9. GitOps.tag_sprint(tag) + push()
```

### 2.2 接口签名速查表

| IF ID | 方法签名 | 输入 | 输出 | 调用方 |
|-------|---------|------|------|--------|
| IF-001 | `DocAnalyzer.load_doc_set()` | — | `Dict[str, str]` | main |
| IF-002 | `DocAnalyzer.analyze_and_decompose(sprint_id: int)` | sprint ID | `List[CodingTask]` | main |

| IF-007 | `Dispatcher.dispatch_batch(tasks: List[CodingTask])` | 批量任务 | `List[TaskResult]` | main |
| IF-008 | `AutoReviewer.review(task, result)` | 任务+结果 | `ReviewResult` | main |
