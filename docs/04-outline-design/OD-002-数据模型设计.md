## §1 核心数据结构

> 映射: MOD-005 (task_models.py, 202 行)

### 1.1 DM-001: TaskStatus 枚举 (11 态)

```python
class TaskStatus(enum.Enum):
    CREATED     = "created"      # 任务已创建，未入队
    QUEUED      = "queued"       # 已入队，等待调度
    DISPATCHED  = "dispatched"   # 已分发到机器，执行中
    CODING_DONE = "coding_done"  # aider 编码完成
    REVIEW      = "review"       # 正在审查
    TESTING     = "testing"      # 正在测试
    JUDGING     = "judging"      # 测试完成，判定中
    PASSED      = "passed"       # ✅ 终态：通过
    FAILED      = "failed"       # 测试失败（可重试）
    RETRY       = "retry"        # 重试等待（将回到 QUEUED）
    ESCALATED   = "escalated"    # ❌ 终态：升级为人工
```

**终态**: PASSED, ESCALATED
**可重试态**: RETRY → QUEUED → DISPATCHED → ...

## §2 状态机设计

> 映射: MOD-009 (state_machine.py, 148 行), FR-014, FR-015

### 2.1 状态转换表 (_TRANSITIONS)

| 当前状态 | 允许的下一状态 | 触发条件 |
|---------|--------------|---------|
| CREATED | QUEUED | `enqueue()` |
| QUEUED | DISPATCHED | `dispatch()` |
| DISPATCHED | CODING_DONE, RETRY, ESCALATED | `coding_done(result)` |
| CODING_DONE | REVIEW | `start_review()` |
| REVIEW | TESTING, RETRY, ESCALATED | `review_done(review)` |
| TESTING | JUDGING | `test_done(test_result)` |
| JUDGING | PASSED, FAILED | `judge(test_result)` |
| FAILED | RETRY, ESCALATED | `handle_failure()` |
| RETRY | QUEUED | `requeue()` |
| **PASSED** | (终态) | — |
| **ESCALATED** | (终态) | — |

### 2.2 状态转换图

```
  CREATED ──enqueue──▶ QUEUED ──dispatch──▶ DISPATCHED
                         ▲                      │
                         │                 coding_done()
                    requeue()              ┌────┤
                         │                 │    ▼
                      RETRY          (fail)│  CODING_DONE
                      ▲  ▲                 │    │
                      │  │                 │ start_review()
              review  │  │  test           │    ▼
              fail    │  │  fail           │  REVIEW
           (retry<3) │  │ (retry<3)       │    │
                      │  │                 │ review_done()
                   REVIEW│               │ ┌───┤
                      ▲  │               │ │   ▼
                      │  FAILED◀─judge──JUDGING◀──test_done──TESTING
                      │                    │
                      │              judge(passed)
                      │                    │
                      │                    ▼
                      │                 PASSED ✅
                      │
                (retry>=3)
                      │
                      ▼
                  ESCALATED ❌
```

### 2.3 重试计数逻辑

| 计数器 | 递增条件 | 判断条件 |
|--------|---------|---------|
| `review_retry` | Review 失败 → RETRY | `total_retries < max_retries` (默认 3) |
| `test_retry` | 测试失败 → RETRY | `total_retries < max_retries` |
| `retry_count` | `requeue()` 时递增 | 记录入队次数 |
| `total_retries` | = `review_retry + test_retry` | 累计不超过 3 次 |

**关键规则**: `total_retries` (review + test) 累计达到 `max_retries` (3) 即 ESCALATED，不分别计数。

### 2.4 TaskStateMachine 查询属性

| 属性 | 类型 | 含义 |
|------|------|------|
| `is_terminal` | bool | PASSED 或 ESCALATED |
| `is_retryable` | bool | 当前状态为 RETRY |
| `is_waiting` | bool | 当前状态为 QUEUED |
| `can_dispatch` | bool | 当前状态为 QUEUED |
| `needs_review` | bool | 当前状态为 CODING_DONE |
| `needs_testing` | bool | 当前状态为 TESTING |

## §3 配置模型

### 3.1 Config 类结构

`Config` 从 `config.yaml` + `.env` 加载，支持环境变量展开 `${VAR}`：

| 配置区 | 键 | 类型 | 默认值 | 说明 |
|--------|---|------|--------|------|
| **orchestrator** | mode | str | "sprint" | sprint / continuous |
| | current_sprint | int | 1 | 当前 Sprint 号 |
| | poll_interval | int | 10 | 轮询间隔 (秒) |
| **project** | name | str | "" | 项目名称 |
| | path | str | "" | 项目路径 |
| | doc_set | Dict | {...} | 文档集 glob 模式 |
| | branch | str | "main" | Git 分支 |
| **llm** | base_url | str | ${OPENAI_API_BASE} | LLM API 地址 |
| | model | str | ${AIDER_MODEL} | 模型名称 |
| | timeout | int | 120 | API 超时 (秒) |
| **machines** | (列表) | List[Dict] | — | 机器配置列表 |
| **task** | max_retries | int | 3 | 最大重试次数 |
| | concurrent_limit | int | 4 | 最大并发编码数 |
| | single_task_timeout | int | 600 | 单任务超时 (秒) |
| **testing** | host | str | "localhost" | 测试执行主机 |
| | pass_threshold | float | 4.0 | 通过阈值 |
| | fallback_threshold | float | 0.8 | 降级容忍阈值 |
| **notification** | dingtalk_webhook | str | ${DINGTALK_WEBHOOK_URL} | 钉钉 Webhook |
| | dingtalk_webhook_secret | str | "" | 签名密钥 |
| | at_mobiles | List[str] | [] | @指定手机号 |
| | at_all | bool | false | @所有人 |

value = re.sub(r'\$\{(\w+)\}', lambda m: os.environ.get(m.group(1), ''), value)

|------|------|
