## 关键实现路径

推荐编码顺序 (依赖排序):

```
Phase 1 — 基础层 (无外部依赖):
  MOD-005 (Reviewer)    → 纯算法, 无 IO
  MOD-012 (Config)      → 所有模块依赖

Phase 2 — 核心层:
  MOD-009 (TestRunner)  → 依赖 Config
  MOD-003 (Machine)     → 依赖 Config
  MOD-004 (TaskEngine)  → 依赖 Machine

Phase 3 — 外部集成:
  MOD-001 (LLMProvider) → 依赖 Config + httpx
  MOD-002 (DocAnalyzer) → 依赖 LLMProvider
  MOD-006 (Issue)       → 依赖 TaskEngine

Phase 4 — 高集成:
  MOD-007 (Dispatcher)  → 依赖 Machine + SSH
  MOD-008 (AutoReviewer)→ 依赖 LLMProvider + Reviewer
  MOD-010 (Reporter)    → 依赖 Config + HTTP
  MOD-011 (GitOps)      → 依赖 Machine + subprocess

Phase 5 — 入口:
  MOD-013 (Main)        → 依赖全部模块
```
